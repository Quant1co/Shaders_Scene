##  Описание

Данный проект представляет собой небольшой 3D-движок, демонстрирующий работу с современным OpenGL (Core Profile). Визуализация включает в себя:

*   **Множественные источники света**: Directional (глобальный), Point (точечный) и Spot (прожектор).
*   **Различные модели затенения**: возможность переключения между **Phong**, **Toon** и **Oren-Nayar** прямо во время выполнения программы.
*   **Камера от первого лица**: свободное перемещение по сцене с помощью клавиш WASD и мыши.
*   **Загрузка 3D-моделей**: использование собственного парсера `.obj` для загрузки геометрии и текстур.
*   **Интерактивность**: возможность включения/выключения фонарика (Flashlight) по нажатию клавиши.

###  Установка и сборка (Visual Studio)

1.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/ваш-username/название-репозитория.git
    ```

2.  **Настройте пути к библиотекам:**
    *   Откройте файл решения `Lab14.vcxproj` в Visual Studio.
    *   Убедитесь, что пути к заголовочным файлам (Additional Include Directories) и библиотекам (Additional Library Directories) в настройках проекта указывают на правильные директории на вашей системе (по умолчанию в проекте используются пути `C:\glm`, `C:\Program Files\glad\include`, `C:\Program Files\SFML-2.6.1\include` и аналогичные для `.lib` файлов).

3.  **Соберите проект:**
    *   Выберите конфигурацию `Debug` или `Release` и платформу `x64`.
    *   Нажмите `Ctrl+Shift+B` для сборки.

4.  **Запустите приложение:**
    *   Нажмите `F5` для запуска с отладкой.
    *   **Важно:** Убедитесь, что исполняемый файл `.exe` и все необходимые шейдеры (`vertex.glsl`, `fragment.glsl`) и модели с текстурами (`plane.obj`, `barrel.obj`, `yellow.jpg` и т.д.) находятся в одной папке.

##  Архитектура и Устройство

Проект построен на базе нескольких ключевых классов:

*   **`Shader`**: Отвечает за загрузку, компиляцию и линковку GLSL-программ. Предоставляет удобные методы для установки uniform-переменных (`setMat4`, `setVec3` и т.д.).
*   **`Camera`**: Реализует логику камеры от первого лица. Хранит векторы положения, направления и "верха". Метод `update` обрабатывает перемещение с помощью клавиатуры.
*   **`Model`**: Управляет загрузкой моделей из файлов `.obj` и текстур. Содержит буферы (VAO, VBO) для отрисовки геометрии. В методе `draw` вызываются соответствующие OpenGL-функции для рендеринга.

###  Система освещения

Вся логика освещения реализована в **шейдерах** (`fragment.glsl`). Это позволяет гибко настраивать параметры без перекомпиляции C++ кода.

*   **Структуры света**: `DirLight`, `PointLight`, `SpotLight` передаются в шейдер как uniform-структуры.
*   **Модели затенения**: В зависимости от значения `shadingModel` (0, 1, 2) фрагментный шейдер применяет одну из функций: `phong`, `toon` или `orenNayar`.
*   **Аттенюация**: Для точечного и направленного источников реализовано затухание света с расстоянием.


##  Используемые технологии

*   **OpenGL 3.3**: Современный графический API.
*   **GLSL (OpenGL Shading Language)**: Для написания вершинных и фрагментных шейдеров.
*   **SFML**: Для создания контекста OpenGL и обработки ввода.
*   **GLM**: Для математических вычислений в C++ коде.
*   **stb_image**: Простая библиотека для загрузки текстур.
